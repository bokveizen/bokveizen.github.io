## Tmux usage
For any command likely to run longer than a brief foreground check, prefer launching it in a new detached `tmux` session rather than running it directly. Use one dedicated session per task with a clear unique name, start it detached in the correct working directory, and run exactly one non-interactive command in that session. Capture stdout and stderr to a persistent log file, and prepend a small execution header that records the start time, session name, working directory, and exact command. Ensure the command's final exit code is written to the log or to a companion status file so completion can be verified without attaching. After launch, always report the session name, exact command, working directory, log path, and how status will be checked. Monitor progress only with non-interactive checks such as `tmux has-session`, `tmux list-sessions`, `tmux capture-pane`, and log tailing. Do not reuse a session for unrelated tasks, do not allow multiple agents to send input to the same session, and do not run interactive programs, pagers, prompts, or editors in detached sessions. Only attach interactively when debugging is explicitly needed. When the task finishes, report whether it succeeded, failed, or is still running based on the recorded exit status, and clean up the session unless there is a reason to preserve it for debugging.
